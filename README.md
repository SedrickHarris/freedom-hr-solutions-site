# Freedom HR Solutions Website

A scalable, SEO-focused, conversion-focused authority website for Freedom HR Solutions, a Las Vegas headquartered HR, payroll, benefits, compliance, ACA reporting, and HR technology consulting firm serving Nevada and nationwide.

Built as a static Next.js export for deployment on Cloudflare Pages.

## Tech Stack

- Next.js (App Router) + TypeScript
- Tailwind CSS v4
- Static export (`output: "export"`) for Cloudflare Pages
- GitHub + Cloudflare Pages

## Local Development

```powershell
npm install
npm run dev      # http://localhost:3000
npm run build    # generates /out
npm run lint
```

The production build outputs a fully static site to `/out`.

## Deployment (Cloudflare Pages)

- Build command: `npm run build`
- Build output directory: `out`
- Root directory: `/`

`/out` is git-ignored and generated by Cloudflare during deploys.

## Architecture

The site is **data-driven**. Page content lives in `src/data` and is rendered through reusable templates, so new pages are added by editing data files rather than writing new route code.

```
src/
  app/            # routes (core pages + dynamic [slug] routes), sitemap.ts, robots.ts
  components/     # ui, layout, sections, cards, forms, seo, templates
  data/           # site, navigation, serviceHubs, services, partners, audiences, locations, faqs, blog, resources
  lib/            # seo, schema (JSON-LD), links (related-page resolver), routes, utils, forms
  types/          # shared content types
docs/site-os/     # client-specific Site OS planning + QA + changelog docs
```

Repetitive page families (service hubs, services, partners, audiences, locations, blog) use dynamic routes with `generateStaticParams`, producing identical static URLs in `/out`.

## SEO

Every page has unique metadata, a single H1, breadcrumbs, internal links, and JSON-LD schema (Organization, WebSite, ProfessionalService, Person, Service, FAQPage, BreadcrumbList, Article). `sitemap.xml` and `robots.txt` are generated at build. The `/thank-you/` page is `noindex` and excluded from the sitemap.

## Forms

`ContactForm` and `AssessmentForm` are front-end only and route to `/thank-you/` on submit. They are structured for future CRM / webhook / GoHighLevel integration: set `NEXT_PUBLIC_LEAD_ENDPOINT` to POST submissions as JSON.

## Brand Preservation Requirement

This project is **not a rebrand**. It preserves the existing Freedom HR Solutions visual identity:

- Current logo direction
- Current red, white, blue, navy, and gray color system
- Current professional, patriotic but restrained HR consulting feel
- Current abstract textured background style where appropriate

It improves architecture, SEO, AEO, GEO, LLM readiness, conversion paths, performance, accessibility, mobile responsiveness, and static-export deployment.

Exact brand hex codes are placeholders aligned to the red/white/blue/navy/gray system and should be confirmed against the current site, logo, or stylesheet. See `docs/site-os/inputs/brand-notes.md`.

## Before Launch

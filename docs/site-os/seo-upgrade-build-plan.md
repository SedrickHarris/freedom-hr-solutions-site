## How Each Page Gets Processed

Every page follows the same three-step cycle:

**Step 1 — Claude Project (strategy):** Generate seed keywords, run Keyword Planner, upload CSV, run pipeline, upload `opportunities.json`. Claude Project analyzes the data and produces the full page brief with keyword map, updated content, AEO FAQs, schema plan, and internal link plan.

**Step 2 — Claude Code (implementation):** Execute the build brief. Update content in data files, update metadata, update schema, update internal links. Run lint, build, confirm `/out`, commit.

**Step 3 — Pass/Fail Gate:** Verify all gate items before marking the page complete.

---

## AI Depth Assignments

- **Level 5 — Beyond-Elite:** Primary revenue pages, competitive keywords, location + service pages
- **Level 4 — Elite:** Hub pages, audience pages, supporting service pages
- **Level 3 — Standard:** Index pages, utility pages with SEO value
- **Level 1 — Utility:** Privacy policy, thank-you — no SEO workflow needed

## Per-Page Workflow (Repeating)

```
1. Claude Project — generate seed keywords for the page
2. Sedrick — run Keyword Planner, export CSV
3. Sedrick — drop CSV into docs/seo/keyword-exports/, run pipeline
4. Sedrick — upload opportunities.json to Claude Project
5. Claude Project — analyze data, produce full page brief
6. Claude Code — implement brief (content, metadata, schema, internal links)
7. Claude Code — lint, build, confirm /out, commit, push
8. Pass/Fail gate — verify and log to changelog
```
